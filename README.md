# home-portal-hub

## 项目简介

home-portal-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，而是作为通用页面集合，便于集中管理与分发静态网页内容。

## 目录说明

```
home-portal-hub/
├── index.html          # 项目入口页面
├── pages/              # 存放独立 HTML 页面的文件夹
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 静态资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本说明文件
```

- `index.html`：仓库根目录的入口页面，可配置为导航或展示所有归档页面链接。
- `pages/`：所有独立 HTML 页面存放于此，每个文件对应一个独立页面。
- `assets/`：存放页面所需的公共样式、脚本和图片等资源，便于统一维护。

## 页面归档说明

本仓库中的每个 HTML 页面均为独立、自包含的静态页面，可单独部署或访问。页面内容不绑定任何特定域名或服务，仅作为归档与发布的载体。

- 页面命名建议清晰、简短，便于识别。
- 所有页面应尽量使用相对路径引用 `assets/` 中的资源，以保证在本地或不同部署环境下均能正常显示。
- 如需修改页面内容，请直接编辑对应 HTML 文件。

## 维护说明

- **新增页面**：在 `pages/` 目录下创建新的 HTML 文件，并按需在 `index.html` 中添加链接。
- **更新页面**：直接编辑对应 HTML 文件，提交后自动更新。
- **删除页面**：移除对应 HTML 文件，并更新 `index.html` 中的链接。
- **资源管理**：公共资源请放入 `assets/` 目录，避免在每个页面中重复引入。
- **版本控制**：建议每次修改后提交清晰的 commit 信息，便于回溯。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 中添加或修改 HTML 文件。
3. 在 `index.html` 中更新页面导航（可选）。
4. 提交并推送至 GitHub，即可通过 GitHub Pages 或其他静态托管服务访问。
